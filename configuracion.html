Configuración inicial del sistema

Sigue estos pasos para activar correctamente tu sistema de reservas ALOTAR y evitar errores frecuentes.

🔐 Calendar ID obligatorio

Este es el calendario donde se guardarán las reservas del cliente.

  1. Abre Google Calendar.
  2. Selecciona el calendario que se usará para las reservas.
  3. Entra a Configuración.
  4. Busca la sección Integrar calendario.
  5. Copia el ID del calendario.

Ejemplo:

abc123@group.calendar.google.com

⚠️ Error común: copiar el nombre del calendario en vez del Calendar ID.

⚠️ Error común: no compartir el calendario con permisos de edición.

✔ El calendario debe estar compartido correctamente para que ALOTAR pueda crear reservas.

🖼 Logo FILE ID

El Logo File ID permite mostrar el logo de tu empresa en la aplicación, en los correos automáticos y en la experiencia visual del sistema.

Requisitos recomendados del logo
  • Formato: PNG preferiblemente.
  • Fondo: transparente.
  • Tamaño ideal: 800 × 800 px.
  • Tamaño mínimo: 400 × 400 px.
  • Peso recomendado: menor a 300 KB.
  • Forma: cuadrado, no tipo banner horizontal.
  • Color: claro, blanco o dorado si se usará sobre fondo oscuro.

✔ Logo recomendado

Logo cuadrado, nítido, con fondo transparente y buena lectura sobre fondo oscuro.

LOGO

✘ Logo no recomendado

Logo tipo banner, pixelado, con fondo blanco o texto demasiado pequeño.

logo muy pequeño / fondo blanco

⚠️ Evita logos con fondo blanco, pixelados, muy oscuros o con texto demasiado pequeño.

Cómo obtener el Logo File ID

  1. Sube tu logo a Google Drive.
  2. Haz clic derecho sobre el archivo.
  3. Selecciona Compartir.
  4. Cambia el acceso a Cualquier persona con el enlace.
  5. Copia el enlace del archivo.

Ejemplo de enlace:

https://drive.google.com/file/d/1ABCxyz123/view

El Logo File ID es esta parte:

1ABCxyz123

⚠️ Si no ves el logo en la vista previa, revisa que el File ID sea correcto y que el archivo esté compartido como “Cualquier persona con el enlace”.

✔ Si el logo está bien configurado, se verá correctamente en la app, en los correos y en la experiencia del cliente.

📍 Configuración de sedes

Cada sede representa un punto de atención físico o virtual donde se podrán crear reservas.

Sede física

  • Nombre de la sede.
  • Dirección completa.
  • Enlace de Google Maps.
  • Capacidad disponible si aplica.

Sede virtual

  • Plataforma: Google Meet, Zoom u otra.
  • Link de acceso.
  • Notas o instrucciones para el usuario.

⚠️ Error común: crear horarios sin haber configurado primero las sedes.

✔ Configura primero las sedes y luego los horarios de atención.

⏰ Horarios de atención

Los horarios permiten que el sistema muestre disponibilidad real para las reservas.

  • Hora de inicio.
  • Hora de finalización.
  • Sede correspondiente.
  • Capacidad por bloque.

⚠️ Sin horarios configurados, no habrá reservas disponibles.

✔ Crea bloques claros y verifica que no se crucen entre sedes.

⚠️ Errores comunes

  • Calendar ID incorrecto.
  • Calendario sin permisos de edición.
  • Logo sin permisos de visualización.
  • Logo muy pesado o con baja calidad.
  • No crear sedes antes de crear horarios.
  • No crear bloques de atención.
