Paso 1: Configuración del sistema

Sigue estos pasos para activar correctamente tu sistema de reservas ALOTAR y evitar errores frecuentes.

Paso 1 de 2 — Configura tu sistema antes de usar el Panel Administrativo.

🔐 Calendar ID obligatorio

El Calendar ID identifica el calendario donde ALOTAR guardará las reservas. Recomendamos usar un calendario exclusivo para reservas y no mezclarlo con calendarios personales.

1. Crea un calendario exclusivo para reservas

  1. Abre Google Calendar.
  2. En el menú izquierdo busca Otros calendarios.
  3. Haz clic en el botón +.
  4. Selecciona Crear calendario nuevo.
  5. Escribe un nombre claro, por ejemplo: Reservas ALOTAR.
  6. Haz clic en Crear calendario.

✔ Esto evita mezclar reservas con eventos personales o internos.

2. Entra a la configuración del calendario

  1. Ubica el calendario que usarás para reservas.
  2. Pasa el mouse sobre el nombre del calendario.
  3. Haz clic en los tres puntos ⋮.
  4. Selecciona Configuración y uso compartido.

3. Copia el Calendar ID

Baja hasta la sección Integrar calendario y copia el campo llamado ID del calendario.

Ejemplo correcto:

abc123@group.calendar.google.com

⚠️ Error común: copiar el nombre del calendario en vez del Calendar ID.

✔ El Calendar ID normalmente termina en @group.calendar.google.com.

4. Comparte el calendario con permisos de edición

Para que ALOTAR pueda crear reservas automáticamente, el calendario debe compartirse con la cuenta indicada por tu asesor o administrador.

  1. En la configuración del calendario busca Compartir con personas o grupos específicos.
  2. Haz clic en Agregar personas o grupos.
  3. Escribe el correo que te indique ALOTAR.
  4. En permisos selecciona Realizar cambios en eventos.
  5. Haz clic en Enviar.

⚠️ Si el calendario no tiene permisos de edición, el sistema no podrá guardar reservas.

✔ Permiso recomendado: Realizar cambios en eventos.

5. Pega el Calendar ID en ALOTAR

Copia y pega el ID exactamente como aparece en Google Calendar. No agregues espacios, comillas ni texto adicional.

✔ Correcto: abc123@group.calendar.google.com

✘ Incorrecto: Reservas ALOTAR

6. Haz una prueba final

Después de guardar la configuración, crea una reserva de prueba y verifica que el evento aparezca en Google Calendar.

  • Si el evento aparece, la configuración está correcta.
  • Si no aparece, revisa el Calendar ID y los permisos.

🖼 Logo FILE ID

El Logo File ID permite mostrar el logo de tu empresa en la aplicación, en los correos automáticos y en la experiencia visual del sistema.

Requisitos recomendados del logo
  • Formato: PNG preferiblemente.
  • Fondo: transparente.
  • Tamaño ideal: 800 × 800 px.
  • Tamaño mínimo: 400 × 400 px.
  • Peso recomendado: menor a 300 KB.
  • Forma: cuadrado, no tipo banner horizontal.
  • Color: claro, blanco o dorado si se usará sobre fondo oscuro.

✔ Logo recomendado

Logo cuadrado, nítido, con fondo transparente y buena lectura sobre fondo oscuro.

LOGO

✘ Logo no recomendado

Logo tipo banner, pixelado, con fondo blanco o texto demasiado pequeño.

logo muy pequeño / fondo blanco

⚠️ Evita logos con fondo blanco, pixelados, muy oscuros o con texto demasiado pequeño.

Cómo obtener el Logo File ID

  1. Sube tu logo a Google Drive.
  2. Haz clic derecho sobre el archivo.
  3. Selecciona Compartir.
  4. Cambia el acceso a Cualquier persona con el enlace.
  5. Copia el enlace del archivo.

Ejemplo de enlace:

https://drive.google.com/file/d/1ABCxyz123/view

El Logo File ID es esta parte:

1ABCxyz123

⚠️ Si no ves el logo en la vista previa, revisa que el File ID sea correcto y que el archivo esté compartido como “Cualquier persona con el enlace”.

✔ Si el logo está bien configurado, se verá correctamente en la app, en los correos y en la experiencia del cliente.

📍 Configuración de sedes

Cada sede representa un punto de atención físico o virtual donde se podrán crear reservas.

Sede física

  • Nombre de la sede.
  • Dirección completa.
  • Enlace de Google Maps.
  • Capacidad disponible si aplica.

Sede virtual

  • Plataforma: Google Meet, Zoom u otra.
  • Link de acceso.
  • Notas o instrucciones para el usuario.

⚠️ Error común: crear horarios sin haber configurado primero las sedes.

✔ Configura primero las sedes y luego los horarios de atención.

⏰ Horarios de atención

Los horarios permiten que el sistema muestre disponibilidad real para las reservas.

  • Hora de inicio.
  • Hora de finalización.
  • Sede correspondiente.
  • Capacidad por bloque.

⚠️ Sin horarios configurados, no habrá reservas disponibles.

✔ Crea bloques claros y verifica que no se crucen entre sedes.

⚠️ Errores comunes

  • Calendar ID incorrecto.
  • Calendario sin permisos de edición.
  • Logo sin permisos de visualización.
  • Logo muy pesado o con baja calidad.
  • No crear sedes antes de crear horarios.
  • No crear bloques de atención.

🚀 Siguiente paso

Una vez completes la configuración inicial, debes ingresar al Panel Administrativo para validar el funcionamiento del sistema y realizar pruebas de reservas.

  • Verifica que las reservas se registren correctamente.
  • Confirma que aparezcan en Google Calendar.
  • Haz una reserva de prueba como si fueras un cliente.
  • Revisa que el enlace público de reservas funcione correctamente.

✔ Cuando esta prueba funcione, tu sistema estará listo para compartirse con tus clientes.

⚠️ No compartas el enlace público antes de validar calendario, sedes y horarios.
